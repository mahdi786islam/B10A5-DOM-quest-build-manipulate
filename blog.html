FAQ about JavaScript

1.What is the Document Object Model (DOM)?

The Document Object Model (DOM) is a programming interface for web documents. It represents the structure of a document (like an HTML or XML document) as a tree of objects, where each node in the tree is an object representing a part of the document.This model is crucial for creating dynamic and interactive web pages, as it enables developers to update the content and structure of a web page without needing to reload the entire page.

3.What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique in JavaScript that leverages event bubbling to handle events at a higher level in the DOM tree rather
than directly on the element where the event occurs. Here's a breakdown of how it works and why it's useful:
How Event Delegation Works
Event Bubbling: When an event occurs on an element, it bubbles up to its parent elements. For example, if you click a button inside a div,
the click event bubbles up from the button to the div.
Delegation: Instead of adding event listeners to multiple child elements, you add a single event listener to a common parent element. This
parent element can then handle events from all its children.
